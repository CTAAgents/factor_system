## 3. 差距详情

### GAP-001: pipeline/ 和 strategies/ 模块无测试

- **模块**: `pipeline/`（`base.py`, `factor_combiner.py`），`strategies/`（`base_v2.py` 部分路径）
- **当前覆盖率**:
  - `pipeline/base.py`: 0%（49 行未覆盖）
  - `pipeline/factor_combiner.py`: 0%（95 行未覆盖）
  - `strategies/base_v2.py`: 51%（76 行未覆盖）
- **影响**: 管线串联、因子组合、策略基类的核心逻辑无法自动回归验证
- **修复方案**: 为 pipeline 模块编写单元测试（至少覆盖基础管线运行、stage 串联、错误处理）；补充 strategies 的策略基类测试
- **预估工作量**: 3-5 个测试文件，~100 个用例

### GAP-002: CLI/监控/调度无测试

- **模块**: `cli.py`, `monitor.py`, `scheduler/tasks.py`
- **当前覆盖率**: 全部 0%
- **影响**: 用户入口和系统运维无可靠性保障
- **修复方案**: 为核心子命令（version, monitor, factor list）编写集成测试；为 scheduler 的 TaskRegistry 编写单元测试
- **预估工作量**: 2-3 个测试文件，~50 个用例

### GAP-003: micro_evolution.py 覆盖率低

- **模块**: `micro_evolution.py`
- **当前覆盖率**: 31%（72 语句中 50 未覆盖）
- **根因**: optuna 依赖在 `[evolution]` extra 中，基础安装不包含；大部分边界条件和异常路径未覆盖
- **修复方案**: 安装 evolution extra 后补充测试，覆盖异常回退路径和参数边界
- **预估工作量**: 1 个测试文件，~30 个用例

### GAP-004: evaluation_chain.py mock 路径未覆盖

- **模块**: `evaluation_chain.py`
- **当前覆盖率**: 90%（134 语句中 14 未覆盖）
- **根因**: 部分 mock 回调路径（模拟 LLM 评估失败）和 edge case 未覆盖
- **修复方案**: 补充 LLM mock 场景的测试覆盖
- **预估工作量**: 1 个测试文件，~10 个用例

| 优先级 | 定义 | 处理时限 | 验证标准 |

## 5. 差距关闭流程

1. 编写测试代码并通过 PR 审查
2. 运行完整测试套件确认全部通过（220 passed, 0 failed）
3. 更新本文件中的差距状态为「已关闭」
4. 更新 `06-testing.md` 中的覆盖统计
5. 如果涉及架构变更，更新 `01-architecture.md`
